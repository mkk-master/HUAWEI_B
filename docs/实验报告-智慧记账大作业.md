## 一、项目概述

本项目基于 HarmonyOS开发了一款"智慧记账"应用，支持日常收支记录、月度预算管理、消费分类统计、端侧 AI 分类预测以及云端消费洞察分析。本次大作业在原应用基础上完成了三项核心升级：引入 C++ Native 财务报表引擎、实现平板分栏布局、编写 HapTest 自动化测试。

### 2.2 核心功能

**（1）收支记账**：支持添加/编辑/删除账单，每条账单包含金额（分存储）、分类、日期、备注四个字段；支持按月筛选、备注关键字搜索、分类过滤。

**（2）预算管理**：按月设置预算额度，首页展示预算执行进度条，实时计算月度剩余/超支情况。

**（3）消费统计**：按月展示消费总额与分类占比，支持饼图视图（Canvas 手绘）和详情列表两种展示模式。

**（4）AI 分类预测**：集成端侧 MindSpore Lite 模型（FastText）做离线分类推理，云端 DashScope (qwen) 做兜底，同时内置本地规则匹配作为双重保险。

**（5）AI 消费洞察**：将当月账单数据汇总后发送 DashScope 生成结构化分析报告（消费结构解读、预算执行情况、超支原因分析、节流建议），并支持多轮自然语言追问。

| `budget` | `id INTEGER PK`, `month TEXT NOT NULL UNIQUE`, `totalBudget INTEGER NOT NULL` | 预算表，按月份唯一约束 |

## 三、任务一：Native C++ 财务报表生成器

### 3.1 设计思路

原应用中，`BillViewModel.buildMonthlyInsightDataText()` 使用 ArkTS/TypeScript 进行账单数据统计（遍历数组、Map 分组、排序、文本拼接）。当账单数量达到数千条时，纯 ArkTS 处理存在性能瓶颈。本次升级将此统计逻辑**完整迁移到 C++ 层**，通过 HarmonyOS NAPI（Native API）实现跨语言调用。

### 3.2 接口设计

```
ArkTS: nativeGenerateMonthlyReport(budgetFen: number, billsJson: string): string
    ↓ 调用 libbillreport.so
C++: generateReport(budgetFen, billsJson) → string
```

参数通过 JSON 序列化传递：前端将 `BillModel[]` 数组 `JSON.stringify` 后传入，C++ 侧用简易 JSON 解析器还原为 `std::vector<BillItem>`。

### 3.3 C++ 侧实现

**文件**：`entry/src/main/cpp/bill_report.h` `/ `bill_report.cpp`

C++ 侧完成以下计算密集型工作：
1. 遍历账单计算总支出（`int64_t` 累加，避免浮点误差）
2. 按分类汇总到 `std::map<std::string, int64_t>`
3. 按金额降序排序（`std::sort` + 自定义比较器）
4. 取前 15 条大额账单
5. 用 `std::ostringstream` 拼装格式化文本报告（含分转元、百分比计算）

**关键代码片段**：

```cpp
// bill_report.cpp 核心逻辑
std::string generateMonthlyReport(int64_t budgetFen, const std::vector<BillItem>& bills) {
    // 1. 计算总支出
    int64_t totalFen = 0;
    for (size_t i = 0; i < bills.size(); i++) {
        totalFen += bills[i].amount;
    }

    // 2. 按分类汇总并排序
    std::map<std::string, int64_t> catMap;
    for (size_t i = 0; i < bills.size(); i++) {
        catMap[bills[i].category] += bills[i].amount;
    }
    std::vector<CategorySummary> catSummaries;
    for (auto it = catMap.begin(); it != catMap.end(); ++it) {
        catSummaries.push_back({it->first, it->second});
    }
    std::sort(catSummaries.begin(), catSummaries.end(), compareCategoryAmount);

    // 3. 拼装格式化报告
    std::ostringstream report;
    report << "账单笔数：" << bills.size() << "\n";
    report << "当月支出合计：" << fenToYuan(totalFen) << " 元\n";
    // ...（详情略）
    return report.str();
}
```

### 3.4 NAPI 桥接层

**文件**：`entry/src/main/cpp/napi_init.cpp`

注册名为 `"billreport"` 的 Native 模块，暴露 `generateReport` 方法：

```cpp
static napi_module billReportModule = {
    .nm_version = 1,
    .nm_register_func = Init,
    .nm_modname = "billreport",
    // ... 通过 __attribute__((constructor)) 自动注册
};
```

参数解析使用 NAPI 标准接口：`napi_get_value_int64`（解析预算数字）和 `napi_get_value_string_utf8`（解析 JSON 字符串）。

### 3.5 ArkTS 侧封装与容错

**文件**：`entry/src/main/ets/utils/NativeReportHelper.ets`

```typescript
import billreport from 'libbillreport.so';
export function nativeGenerateMonthlyReport(budgetFen: number, billsJson: string): string {
  return billreport.generateReport(budgetFen, billsJson);
}
```

**容错设计**：`BillViewModel.buildMonthlyInsightDataText()` 优先调用 Native 路径，若 `libbillreport.so` 加载失败或被 throw，自动回退到 ArkTS 纯逻辑，保证功能不中断：

```typescript
try {
  const nativeResult = nativeGenerateMonthlyReport(budgetFen, billsJson);
  if (nativeResult && nativeResult.length > 0) {
    return `月份：${month}\n${nativeResult}`;
  }
} catch (e) {
  console.error('Native 报表生成失败，回退到 ArkTS 逻辑');
}
return this.buildMonthlyInsightDataTextFallback(month, bills, budgetFen);
```

### 3.6 构建配置

**CMakeLists.txt**：
```cmake
add_library(billreport SHARED napi_init.cpp bill_report.cpp)
target_link_libraries(billreport PUBLIC libace_napi.z.so libhilog_ndk.z.so)
```

## 五、开发过程中遇到的问题及解决方法

### 5.1 C++ 编译问题：结构体前置引用

**现象**：`ninja` 编译报错 `member access into incomplete type 'const struct CategorySummary'`。

**原因**：`compareCategoryAmount` 函数在 `CategorySummary` 结构体定义之前引用了其成员。

**解决**：将 `struct CategorySummary { ... }` 的定义移到 `compareCategoryAmount` 之前。

### 5.2 ArkTS 严格类型检查：对象字面量

**现象**：ArkTS 编译器报错 `Object literal must correspond to some explicitly declared class or interface`。

**原因**：ArkTS 不允许无类型的对象字面量，`bills.map(b => ({ amount: b.amount, ... }))` 中的字面量缺少类型声明。

**解决**：声明 `interface BillJsonItem`，显式标注类型：

```typescript
interface BillJsonItem {
  amount: number; category: string; date: string; remark: string;
}
const items: BillJsonItem[] = bills.map((b: BillModel): BillJsonItem => {
  return { amount: b.amount, category: b.category, date: b.date, remark: b.remark } as BillJsonItem;
});
```

### 5.3 模拟器 ABI 不匹配

**现象**：HAP 部署时报错 `install parse native so failed - Abi type does not match`。

**原因**：`abiFilters` 默认只编译了 `arm64-v8a`（真机），模拟器是 `x86_64`。

**解决**：在 `build-profile.json5` 的 `externalNativeOptions` 中添加 `"abiFilters": ["arm64-v8a", "x86_64"]`。

### 5.4 饼图切换月份不重绘

**现象**：StatisticsPage 切换月份后饼图不更新。

**原因**：`PieChart` 组件使用 Canvas，`onReady` 回调只在首次挂载时触发，数据变化不会自动重绘。

**解决**：两处联动——① `PieChart` 内部添加 `@Watch('onDataChange')` + `canvasReady` 标志，数据变化时主动调用 `drawPieChart()`；② 父组件 `StatisticsPage` 给 `PieChart` 加上 `.key(\`pie-${this.refreshFlag}\`)` 强制更换实例。

## 六、跨语言与跨设备代码实现与效果

### 6.1 跨语言（ArkTS ↔ C++）

| 层级 | 文件 | 职责 |
|------|------|------|
| ArkTS 入口 | `BillViewModel.ets:367-373` | 序列化数据、发起 Native 调用 |
| ArkTS 封装 | `NativeReportHelper.ets` | `import libbillreport.so`，暴露函数 |

**容错机制**：Native 加载失败自动回退到纯 ArkTS 实现，确保在任何环境（预览器、模拟器、真机）下功能可用。
