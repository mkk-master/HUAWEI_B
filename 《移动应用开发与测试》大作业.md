# 《移动应用开发与测试》大作业

## 1. 作业概述

本次作业要求同学们以**小组**为单位，完成开发升级和测试任务。通过引入**底层原生代码（C/C++/Rust）**提升核心性能，并利用 **HarmonyOS 的分布式特性**实现多设备适配；以及使用HapTest测试本小组自己完成的App。

- **核心目标**：掌握 ArkNDK 开发流程（Native 侧逻辑），理解声明式 UI 的响应式布局（多端适配），实现“一次开发，多端部署”的工业级标准。理解自动化测试流程和测试方法。



## 2. 核心升级开发任务

### 任务一：原生底层增强 (Native Engine Upgrade)

**要求**：将应用中的某个“核心逻辑”或“计算密集型任务”迁移到 Native 侧（使用 C/C++ 或 Rust），通过**NAPI** 与 ArkTS 通信。

1. **算法增强**：如财务应用的加密算法、商城应用的搜索排序算法、笔记应用的全文索引、或图片处理应用中的像素级滤镜。
2. **安全性**：使用 C/C++ 实现敏感数据（如用户秘钥、支付信息）的混淆与安全存储逻辑。


### 任务二：全场景适配 (Cross-Device Adaptation)

**要求**：应用必须能够完美运行在**手机、平板**两种以上的设备形态上，且不能是简单的等比例拉伸。

1. **响应式布局**：实现不同屏幕下的不同排版。
2. **分栏架构**：在平板/电脑端必须实现“列表-详情”双栏布局（SideBarContainer 或 Navigation 的多栏模式）。
